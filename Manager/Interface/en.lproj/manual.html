1. What is "Localization Manager"?
 

Localization Manager is tool for developers that helps to simplify the process of localizing an application. It's part of the blue:tec Localization Suite, available under www.blue-tec.com.

Localization Manager's job in this suite is to control the localization process, creating and reading the localization (which can be edited using the "Localizer" application) and the interface files. More on this in the next section.

2. How to use it
  2.1 Adding languages
   

Once you created a new project by choosing "New" from the "File" menu, click on the plus button below the left table or choose "Add Language" in the "Localization"->"Language" menu. Then select the languages you want to have your application localized in. In our case, we chose English as our main language and German.

back to top
 
  2.2 Adding files
   

Before adding file, we first have to set a reference language. This is the language you make all changes in. In our case, let's choose English.

Now click on the plus below the right table to add files. Optionally drag and drop them onto the table. Supported files are: .strings, .nib, .rtf, .rtfd, .txt.

Files that are not checked off won't be included in the later created .loc file(s). You may want to use this to disable some files that either haven't changed or shouldn't be translated by localizers (e.g. a .strings file containing urls or paths).

back to top
 
  2.3 File details
   

Select a file in the right table and click on the button with the magnifier. This will bring up a new window where you can see what's actually in the file.

In addition, you can set two states: Active and updated. If active is not checked off, the appropriate key won't be included in the .loc file. If the second checkbox is checked off, the key will be marked as updated in the .loc file.

  2.4 Updating and resetting a language/file
   

Update Files

Checks which files have been changed in the reference language and updates them in all languages.

This is the most common button and so the one most optimized. This checks all reference files for whether there have been updates and imports all the changes. Then just the changed files are updated. (For details on this, see the description of "Update File".)

You might want to use "Update Files" it before creating .loc files and after importing them, after having made some changes to the interface and so on. You should rarely need to use the other buttons below as they're very specific and "Updates Files" is more or less a merge of all of them.

Update Language

Use this button to regenerate all localized files of the selected languages.

This uses the interface data of the formerly localized and adjusted files, compares them to the recent version of the same files in the reference language and then recreates them using the old interface but retaining all changes made to the reference language. You might want to use this after importing a .loc file to tell the application that it should write the localized data to disk.

Reset Language

Use this button to remove all localized files of the selected languages and then recreate them from the reference.

This removes all files of the selected languages and then recreates them by copying the reference languages' files and using the translated strings to translate the interface. You might want to use this to delete the interface of the selected language and then readjust it, maybe because it has been corrupted.

Update File

Use this button to regenerate all localized versions of the selected file.

This uses the interface data of the formerly localized files, compares them to the recent version of the same file in the reference language and then recreates them using the old interface but retaining all changes made to the reference language. You might want to use this after updating a file.

Reset File

Use this button to removes the file in all languages and recreate it.

This removes all localized versions of the selected files and then recreates them by copying the reference languages' files and using the translated strings to translate the interface. You might want to use this to delete the interface of a selected file and then readjust it, maybe because it has been corrupted.

  2.5 Create localization data
   

Choose "Save Localization Files..." from the "Localization" menu to save the Localizer (.loc) files. Localizer files are the ones being sent to translators. So this is one of the most important steps in the localization process.

You may want all languages to be saved into one file, e.g. for you to check the consitency of of various patterns and so. Selecting "all languages in separate files" will - as it says - create a separate file for each language. These files will be smaller than the composite one but having many files will require a some more logistics.

Including an Interface Preview will allow your localizers to see the changes they make in a more or less good preview. This will show the window / view the string is contained in and will then replace the reference string by the localized. Please tell you localizers that they don't have to try to match this interface preview as there always remains the ability to adjust the size of controls.

You'll find more detailed instructions how to use Localizer in its help.

  2.6 Read localized data
   

Choose "Open Localization Files..." from the "Localization" menu and open the Localizer (.loc) files. This is also one of the most important steps in the localization process as here the localizations get matched to your database.

Importing a .loc file just imports the keys/strings but doesn't create any files. You'll have to use the "Update Files" button to achieve this. It's also possible to use one of the others like "Update language" but in fact "Update Files" is the one best suited. Please also notice that Localizer files are key-based. This means that if you somehow recreate the .ldb file you're still able to use the same .loc files as at least not all keys will change. (To do this you have to use the "Import Localization Files..." command from the Localization menu. For more see section 3.)

  2.7 Tips
   

- There's no need to update the localization database every time you change the interface. Just work on your project as long as it takes and when you achieved a state where there will be no more or just very few changes to be done to the interface you start your work on the localization. Localizer files just have to be created when you want to change something or to send them to your translator.

- You might also find the "updated" and "active" key flags useful. They can be set/unset in the file details windowdow (see section 2.3). Using the "active" flag you can deactivate several keys that don't have to be localized (e.g. patterns, URLs or date formaters). The "updated" flag might also be helpful for your localizers to get the job done. Localizer will read "updated" flags as "changed", making it easy to find the updated sections. You might find it useful that the "updated" flag is set automatically when using "Update Files" and there have been changes. Also notice that you have to uncheck it manually. It's suggested to uncheck after importing all .loc files or after a release (which should be nearly the same).

3. The "Localization" menu
  Update Files
  Works like the "Update Files" button, see section 2.4.
  Read Files
  This does just a part of the "Update Files" command - it just imports the files but does not write the localized versions. You may want to use this in combination with the "Update File" or "Update Language" commands, e.g. when you updated reference files but don't want all files to be updated.
  Languages menu
  These do the very same thing as the buttons below the languages list. The item "Add Custom Language..." allows you to add a language that's not in the default list when using "Add Language...". See section 2.4 for more details.
  Files menu
  These do the very same thing as the buttons below the files list. See section 2.4 for more details.
  Save / Open Localization Files...
  See sections 2.5 and 2.6.
  Import existing Languages / Files...
  Very important commands that allow you to move your existing projects to the database or to import existing localized files. "Import existing Languages..." requires you to select .lproj folders, "Import existing Files..." requires you to select files within a .lproj folder (this is to determine the language of the file). Both commands create an new entry in the languages list if the imported languages do not yet exist.
  Import Localization Files...
  Useful command which works very similar to "Open Localization Files..." but does not just import changed but all keys. This one is usefull when saving your whole database to a Localizer (.loc) file and later reimporting it for whatever reason.
